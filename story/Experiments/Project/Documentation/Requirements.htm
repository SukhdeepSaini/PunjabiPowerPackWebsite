Project Requirements

Functional Requirements

  • login and account : Currently there is only one active account and the feature for creating additional account is disabled as per the instructors notice. I have written the specified function for registering new user and this functionality can be added in future. User can only make purchase of items after successfull login.All other features of the website can be accessed as a guest user and does not require login.

  • Purchase of items : Users should be successfully able to purchase single or multiple products using the website.

  • Get Workout routines : Users should be able to successfully check the information on workout routines using the website. They should be able to customize their plans as per their requirements.

  • Diet Information and Recipes : User can get healthy food recipes using the website and can get information on the nutrients present in each of the recipe.

  • Able to read fitness Articles : User should be able to read the fitness motivational articles on website by using the search facility.

  • Watch Videos : Users should be able to watch fitness motivational videos and excercise videos on the website.

Non Functional Requirements

  • Availability : The website should be readily available to users whenever they try to log in to the account or trying accessing it as a guest user. Availability is one of the very obvious yet very important requirements for any system. Users can try to make a purchase or check their workout schedule at any point of time which implies that the data stored on the server should always be available.

  • Multiple Users : The system should be able to handle multiple user logins synchronously.Multiple user logins at once should not make a difference in the functionality of the website.

  • Consistency of data : Data for the different products displayed for selling should remain consistent when user buys a single or multiple products.

  • Correctness and Verification : The workout routines and healthy diet options data displayed on the website should be checked for correctness.The data should be displayed after proper consultation from a dietician and Nutritionist.

Use Cases

    Post Conditions : User is able to login successfully.
    Step by step :
    User : User is provided with the valid credentails to login by the Admin.
    System : Validates the information
    User : Submit the credentails informtion
